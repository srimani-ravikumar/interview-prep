5. Firewalls

Types

  1. Packet Filtering üü¢
    • Network layer; static rules; low overhead.
    • Pros: Simple, fast
    • Cons: Limited inspection, spoofing vulnerability
    • Use: Small networks, edge devices
  1. Stateful Inspection üîµ
    • Transport layer; tracks connection state.
    • Pros: Advanced security, dynamic traffic handling
    • Cons: Resource intensive, potential latency
    • Use: Enterprise networks, high-security systems
  1. Application Layer Firewalls üñ•Ô∏è
    • Inspects payload, understands application protocols.
    • Pros: Detects sophisticated attacks (SQLi, XSS)
    • Cons: Slower, high resource usage
    • Use: Cloud apps, enterprise, e-commerce

Takeaway: Firewall selection depends on network complexity and security needs.

6. Intrusion Detection & Prevention Systems (IDPS)

  • IDS ‚Äì Passive monitoring, alerts only.
  • IPS ‚Äì Active, blocks malicious activity.

Detection Techniques

  • Signature-Based ‚úÖ ‚Äì Matches known attack patterns.
  • Anomaly-Based üîç ‚Äì Detects deviations from normal behavior.
  • Hybrid ‚ö° ‚Äì Combines both for improved accuracy.

Takeaway: IDPS enhances network visibility and threat mitigation.

7. Virtual Private Networks (VPNs)

  • Purpose: Secure, encrypted connection over public/private networks.
  • Types: Remote Access, Site-to-Site, SSL/TLS, IPSec
  • Pros: Privacy, remote access, anonymity
  • Cons: Performance overhead, reliance on provider security, misconfigurations

Takeaway: VPNs secure communication over untrusted networks.

8. Public Key Infrastructure (PKI)

  • Components: CA, RA, public/private keys, digital certificates, CRL
  • Functions: Authentication, encryption, integrity
  • Applications: Secure email, HTTPS, digital signatures, VPN access

Takeaway: PKI is the backbone of digital trust and secure authentication.

9. Cryptography

Types

  • Symmetric Encryption üîë ‚Äì Same key for encryption & decryption (AES, DES). Fast but key distribution challenge.
  • Asymmetric Encryption üîë ‚Äì Public/private key pair (RSA, ECC, DSA). Secure key exchange but slower.
  • Hybrid Encryption ‚ö° ‚Äì Combines both (TLS, PGP) for speed and security.

Core Objectives: Confidentiality, Integrity, Authentication, Non-repudiation

Takeaway: Cryptography ensures secure communication, data integrity, and identity verification.

10. TLS/SSL

  • Purpose: Secure network communication; TLS is modern standard.
  • Key Features: Encryption, Authentication, Integrity
  • Handshake Process: Hello ‚Üí Certificate ‚Üí Key Exchange ‚Üí Encrypted Communication ‚Üí Termination
  • Versions: SSL 2.0/3.0 (deprecated), TLS 1.0/1.1 (deprecated), TLS 1.2 (widely used), TLS 1.3 (current)
  • Applications: HTTPS, VPNs, secure emails, online payments
  • Best Practices: Use TLS 1.3, strong keys (2048-bit RSA/ECC), disable deprecated protocols, implement HSTS

Takeaway: TLS/SSL protects web traffic and sensitive communication from interception.
