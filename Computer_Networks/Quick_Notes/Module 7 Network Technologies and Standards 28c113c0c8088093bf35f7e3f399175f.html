1. Ethernet Technology and Standards (IEEE 802.3) üåê

Definition:

Ethernet is a wired LAN technology defined by IEEE 802.3, providing high-speed, reliable communication using MAC addresses over physical media.

Key Features:

  • Operates at Layer 1 & 2 (Physical + Data Link).
  • Unique MAC addresses for device identification.
  • Supports speeds from 10 Mbps ‚Üí 400 Gbps.

How It Works:

  • Frames: Contain source/destination MAC, payload, CRC for error detection.
  • CSMA/CD: Prevents collisions in shared networks.
  • Transmission Media: Twisted pair, coaxial, fiber optics.

3. Ethernet Switching & VLANs üîÄ

Ethernet Switching

Definition:

Switches connect devices in a LAN using MAC addresses, forwarding frames intelligently.

How It Works:

  • MAC Table: Maps addresses ‚Üí ports
  • Frame Forwarding:
    • Unicast ‚Üí single device
    • Broadcast ‚Üí all devices
    • Multicast ‚Üí device group
  • Learning: Automatically populates MAC table
  • Loop Avoidance: STP prevents broadcast storms

Types:

  • Unmanaged: Plug-and-play, small networks
  • Managed: VLANs, QoS, monitoring; enterprise-grade

Advantages:

  • Efficient, secure, scalable

Applications:

  • Homes, offices, data centers

One-liner takeaway: ‚ö° Switches = Efficient traffic management in LANs.

VLANs (Virtual LANs)

Definition:

Logical segmentation of a LAN into isolated broadcast domains for performance & security.

How It Works:

  • Segmentation: Confines broadcast traffic to VLAN
  • Tagging: IEEE 802.1Q tags identify VLAN traffic
  • Inter-VLAN Routing: Requires Layer 3 device

Types of VLANs:

  • Default VLAN, Data VLAN, Voice VLAN, Management VLAN

Advantages:

  • Improved security, performance, and simplified management

Applications:

  • Enterprise networks, public spaces, educational institutions

One-liner takeaway: ‚ö° VLANs = Logical isolation for secure & efficient LANs.

Interview Focus üîç:

  • VLAN tagging, inter-VLAN routing
  • Differences between unmanaged & managed switches

4. Data Link Protocols (Layer 2) üîó

1. Point-to-Point Protocol (PPP)

Definition:

Protocol for direct node-to-node communication, supports multiple network layer protocols.

Key Features:

  • Encapsulation (IPv4/IPv6)
  • Authentication: PAP, CHAP
  • Error Detection: CRC
  • Multilink PPP (MLPPP) for higher throughput

Applications:

  • Dial-up & DSL connections

One-liner takeaway: ‚ö° PPP = Reliable point-to-point connectivity.

2. High-Level Data Link Control (HDLC)

Definition:

Bit-oriented protocol for synchronous communication, ensuring error-free transmission.

Key Features:

  • Frames: Flag, Address, Control, Payload, CRC
  • Modes:
    • NRM ‚Üí Master-slave
    • ABM ‚Üí Peer-to-peer

Applications:

  • Leased lines, industrial networks

One-liner takeaway: ‚ö° HDLC = Reliable synchronous link communication.

3. Spanning Tree Protocol (STP)

Definition:

Prevents loops in Ethernet networks, maintaining a loop-free topology.

How It Works:

  • Root Bridge Election: Lowest Bridge ID becomes root
  • Path Cost Calculation for shortest route
  • Port Roles: Root, Designated, Blocked
  • Topology Updates: BPDUs adapt dynamically

Applications:

  • Enterprise networks, data centers

One-liner takeaway: ‚ö° STP = Ensures loop-free, stable Ethernet networks.

Interview Focus üîç:

  • Differences between PPP, HDLC, STP
  • Frame structure & port roles
  • Use cases for STP
