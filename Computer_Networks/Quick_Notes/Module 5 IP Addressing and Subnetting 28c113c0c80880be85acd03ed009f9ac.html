1.2 IPv6 (Internet Protocol v6) üåê

Structure:

  • 128-bit addresses in colon-hexadecimal notation (e.g., 2001:db8::1)
  • 8 segments, supports shorthand for zeros

Header Fields:

  • Version, Traffic Class, Flow Label
  • Payload Length, Next Header, Hop Limit
  • Source & Destination Address

Key Features:

  • Large address space (~340 undecillion)
  • Built-in IPsec security üîí
  • Auto-configuration (SLAAC)
  • Efficient routing

Takeaway: IPv6 solves IPv4 limitations, ideal for IoT and modern networking.

2. Subnetting & Supernetting üè∑Ô∏è

2.1 Subnetting

Definition: Dividing a large network into smaller subnets for better management.

Purpose:

  • Reduce network congestion üìâ
  • Enhance security üîí
  • Optimize IP allocation

Example:

  • Original Network: 192.168.1.0/24 (254 usable IPs)
  • Subnets:
    • 192.168.1.0/26 ‚Üí 64 IPs
    • 192.168.1.64/26 ‚Üí 64 IPs

Takeaway: Subnetting improves efficiency and security in LANs.

2.2 Supernetting

Definition: Combining multiple smaller networks into a larger network.

Purpose:

  • Reduce routing table size üóÇÔ∏è
  • Optimize routing efficiency
  • Simplify network management

Example:

  • 192.168.0.0/16 aggregates 192.168.0.0/24 and 192.168.1.0/24

Takeaway: Supernetting simplifies routing in large networks, commonly used by ISPs.

3. NAT, DHCP, and DNS üõ†Ô∏è

3.1 Network Address Translation (NAT)

Definition: Modifies IP addresses in packets between private and public networks.

Types:

  • Static NAT: 1:1 mapping (e.g., server hosting website)
  • Dynamic NAT: Pool-based mapping
  • PAT (NAT Overload): Multiple private IPs ‚Üí 1 public IP (home networks)

Advantages:

  • Conserves public IPs
  • Hides internal network structure
  • Enables Internet access

Limitations:

  • Adds latency
  • Compatibility issues with direct-connect protocols

Takeaway: NAT enables multiple devices to share a single public IP while improving security.

3.2 Dynamic Host Configuration Protocol (DHCP)

Definition: Automates IP assignment and configuration.

Process:

  1. Discovery ‚Üí 2. Offer ‚Üí 3. Request ‚Üí 4. Acknowledgment

Components:

  • DHCP Server, DHCP Client, Lease Time

Advantages:

  • Automates IP management
  • Reduces configuration errors
  • Scalable for large networks

Takeaway: DHCP simplifies network management and avoids manual errors.

3.3 Domain Name System (DNS)

Definition: Resolves domain names to IP addresses.

Process:

  1. Query ‚Üí 2. Recursive Resolver ‚Üí 3. Authoritative Server ‚Üí 4. Response

Record Types:

  • A Record: IPv4
  • AAAA Record: IPv6
  • CNAME: Aliases
  • MX: Mail servers

Advantages:

  • User-friendly navigation üåê
  • Scalable & redundant
  • Supports Internet services

Limitations:

  • Vulnerable to attacks (spoofing, cache poisoning)

Takeaway: DNS makes the internet navigable and scalable for millions of users.
