Definition:

Routing algorithms are mathematical methods routers use to determine the optimal path for data packets. They consider network topology, traffic, and link costs.

Purpose:

  • ‚úÖ Ensure efficient data delivery
  • ‚úÖ Optimize network resource usage
  • ‚úÖ Enable load balancing in dynamic networks

Types of Routing

A. Static Routing

  • Configuration: Manually configured by network admins
  • Key Features:
    • Predictable routes
    • No overhead for route updates
    • Manual updates required on topology change
  • Advantages:
    • Simple to configure in small networks
    • Conserves bandwidth
    • Enhances security
  • Disadvantages:
    • Not scalable
    • Cannot adapt to network failures
  • Use Case: Small offices, home networks

Takeaway: Best for small, stable networks with predictable traffic. üîç

B. Dynamic Routing

  • Configuration: Automatically updates routing tables using protocols
  • Key Features:
    • Adaptable to topology changes
    • Periodic updates between routers
    • Uses protocols like RIP, OSPF, EIGRP
  • Advantages:
    • Scalable for large networks
    • Resilient to failures
    • Reduces manual admin effort
  • Disadvantages:
    • Consumes bandwidth for updates
    • More complex setup
  • Use Case: Enterprises, ISPs, dynamic large networks

Takeaway: Ideal for large, dynamic networks needing automated route management. üîç

2Ô∏è‚É£ Routing Protocols

Routing protocols determine how routers exchange information and select paths.

Categories

  1. Interior Gateway Protocols (IGPs): Routing within a single AS
    • Examples: RIP, IGRP, EIGRP, OSPF
  1. Exterior Gateway Protocols (EGPs): Routing between ASs
    • Example: BGP

A. Distance Vector Protocols (DV)

  • Mechanism: Share entire routing tables with neighbors
  • Features:
    • Periodic updates
    • Metric-based (hop count) path selection
    • Decentralized decision-making
  • Pros: Simple, low resource usage
  • Cons: Slow convergence, count-to-infinity problem
  • Protocols: RIP, EIGRP

Takeaway: Best for small networks with limited resources. üîç

B. Link State Protocols (LS)

  • Mechanism: Build a complete network map and calculate shortest paths
  • Features:
    • Global topology awareness
    • Event-driven updates
    • Shortest path calculation (Dijkstra)
  • Pros: Fast convergence, scalable, accurate routing
  • Cons: Resource-intensive, complex configuration
  • Protocols: OSPF, IS-IS

Takeaway: Suited for large, dynamic networks requiring speed and accuracy. üîç

3Ô∏è‚É£ Switching Techniques

Switching defines how data flows between devices in a network.

Purpose:

  • Efficient data transfer ‚ú®
  • Optimized resource use ‚ú®
  • Support for real-time & asynchronous communication ‚ú®

A. Circuit Switching

  • Mechanism: Establishes a dedicated path before communication
  • Features: Continuous connection, reserved bandwidth
  • Pros: Reliable, guaranteed bandwidth
  • Cons: Inefficient resource use, high setup time
  • Applications: Telephone networks, video conferencing

Takeaway: Best for real-time, guaranteed communication. üîç

B. Packet Switching

  • Mechanism: Data divided into packets; routed independently
  • Features: Dynamic path, stateless communication
  • Pros: Efficient, scalable, resilient
  • Cons: Out-of-order delivery, variable delay
  • Applications: Internet, streaming, online gaming

Takeaway: Backbone of modern data networks. üîç

C. Message Switching

  • Mechanism: Entire message stored and forwarded at intermediate nodes
  • Features: Store-and-forward, no dedicated path
  • Pros: Efficient for non-real-time, handles congestion
  • Cons: High latency, storage requirements
  • Applications: Email, file sharing

Takeaway: Suited for non-real-time communication. üîç
