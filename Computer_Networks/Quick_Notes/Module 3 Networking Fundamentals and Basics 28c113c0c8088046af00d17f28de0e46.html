1. Network Cabling and Connectors üîå

Definition:

Physical mediums (cables) and connectors that enable data transfer between devices in a network.

Importance:

  • Provides the foundation for network communication
  • Determines speed, reliability, and scalability

1.1 Twisted Pair Cables

  • Structure: Two insulated copper wires twisted together to reduce electromagnetic interference (EMI)
  • Types:
    • UTP (Unshielded): Cost-effective, widely used in Ethernet
    • STP (Shielded): Reduces EMI, ideal for industrial environments
  • Connectors: RJ45 (Ethernet), RJ11 (telephone)
  • Pros: Easy installation, flexible, cost-effective
  • Cons: Susceptible to interference, limited bandwidth

1.2 Fiber Optic Cables

  • Structure: Glass/plastic fibers transmitting data as light
  • Types:
    • Single-mode (SMF): Long-distance, single light wave
    • Multi-mode (MMF): Short-distance, multiple light waves
  • Connectors: LC, SC, ST
  • Pros: High bandwidth, immune to EMI, long distance
  • Cons: Expensive, fragile

1.3 Coaxial Cables

  • Structure: Central conductor + insulation + metallic shield + outer jacket
  • Connectors: BNC, F-type
  • Pros: EMI resistant, durable
  • Cons: Limited speed, bulky

2. Network Devices üñß

Definition: Hardware enabling communication and data transfer between devices.

Importance:

  • Facilitates routing & switching
  • Enhances performance & scalability
  • Maintains data integrity

2.1 Router

  • Connects multiple networks, routes packets using IP addresses
  • Functions: Best path selection, internet access, NAT/firewall security
  • Use Case: Home/office internet access

2.2 Switch

  • Connects multiple devices within a LAN, forwards data using MAC addresses
  • Use Case: LAN communication between computers, printers, servers

2.3 Hub

  • Broadcasts data to all connected devices
  • Layer: Physical
  • Limitation: Causes network congestion
  • Use Case: Small/simple networks

2.4 Bridge

  • Connects multiple networks into a single network
  • Filters traffic using MAC addresses
  • Use Case: Expanding LANs with segmentation

2.5 Repeater

  • Amplifies & regenerates signals to extend network range
  • Use Case: Long-distance communication (e.g., buildings, Wi-Fi extension)

Device Comparison:

5. Network Access Control (NAC) üîê

Definition: Security solution to regulate device/user access based on identity and compliance.

Purpose:

  • Authenticate & authorize devices
  • Ensure compliance with security policies
  • Monitor devices for ongoing threats

5.1 How NAC Works

  • Authentication: Verify identity (passwords, certificates, biometrics)
  • Authorization: Assign access levels based on roles or context
  • Compliance Check: Verify antivirus, patches, firewalls
  • Enforcement: Continuous monitoring, isolate non-compliant devices

5.2 Types

TypeTimingFocusAction
Pre-AdmissionBefore network accessCompliance/authenticationDeny/quarantine non-compliant devices
Post-AdmissionAfter accessMonitoring/detectionBlock/isolate suspicious devices

5.3 Benefits

  • Enhanced security
  • Improved visibility of devices
  • Automated threat response
  • Regulatory compliance (GDPR, HIPAA, PCI-DSS)

‚ú® Takeaway: NAC ensures secure, policy-compliant network access.

üîç Interview Focus: Pre vs post admission NAC, components, and enforcement mechanisms.
