Module 2: Process Management

1️⃣ Process and Its States ✨

Definition:

  • A process is a program in execution—an active entity consuming CPU, memory, and I/O resources.
  • Each process has a Process Control Block (PCB) storing its essential information.

Real-Life Analogy: 🍳

  • A recipe = program, chef = CPU, ingredients/tools = resources.
  • The process executes steps sequentially, requiring specific resources at each stage.

Process in Memory:

  • Text Segment: Executable code.
  • Data Segment: Global/static variables.
  • Heap: Dynamic memory allocation.
  • Stack: Local variables, function calls.

Process States:

StateDescriptionKey Points
NewProcess is being createdPCB allocated
ReadyWaiting for CPUManaged in a queue
RunningExecuted by CPUOne process per core
WaitingWaiting for I/OReturns to ready once done
SuspendedTemporarily paused/swappedSaves resources
TerminatedCompleted executionResources released

State Transitions:

  • New → Ready → Running → Waiting → Ready → Running → Terminated
  • Scheduler and I/O operations trigger transitions.

One-Liner Takeaway:

Processes move through states to allow efficient multitasking and CPU utilization.

Interview Focus 🔍:

  • Difference between ready, waiting, and suspended.
  • Life cycle of a process.

2️⃣ Process Control Block (PCB) ⚙️

Definition:

  • PCB is a data structure storing all information about a process.
  • Acts like an identity card for the OS to manage scheduling, execution, and termination.

Information in PCB:

  • Process state, PID, program counter, CPU registers.
  • Memory management info (base/limit, pages, segments).
  • I/O status info, CPU usage, priority, accounting.

Importance:

  • Context switching: Saves/resumes process state.
  • Scheduling decisions: Determines next process.
  • Resource allocation & error handling.

Advanced Perspective:

  • May include security info, privileges, and multiprocessor scheduling data.

One-Liner Takeaway:

PCB is the backbone that allows the OS to control, schedule, and switch processes efficiently.

Interview Focus 🔍:

  • Components of PCB, role in context switching.

3️⃣ Context & Context Switching 🔄

Definition:

  • Context: CPU/system state required to execute a process.
  • Includes registers, process state, memory info, etc.

Context Switching:

  • Switching CPU from one process to another to enable multitasking.

Analogy: 🧑‍🏫

  • Teacher grading multiple students’ assignments, keeping notes of where they left off.

Events During Context Switching:

  1. Interrupt or system call.
  1. Save current process state in PCB.
  1. Load next process state from PCB.
  1. Resume execution.

Components Involved:

  • PCB, CPU registers, interrupts, scheduler.

Advantages:

  • Efficient CPU utilization, multitasking, fairness.

Disadvantages:

  • Overhead, performance impact, OS complexity.

One-Liner Takeaway:

Context switching allows multitasking but adds overhead.

Interview Focus 🔍:

  • Steps in context switching, overhead, and its necessity.

4️⃣ Kernel Mode vs User Mode 🔒

Definition:

  • Dual-mode operation: Kernel Mode (privileged) & User Mode (restricted).

Analogy: 🏦

  • Bank: public area = user mode, restricted area = kernel mode.

User Mode:

  • Runs user applications with limited permissions.
  • Errors do not crash the system.

Kernel Mode:

  • Executes OS core functions.
  • Full access to CPU, memory, I/O.
  • Errors can crash the system.

Mode Change Events:

  • System calls, interrupts, faults.

Mode Bit:

  • Hardware bit to distinguish user (1) vs kernel (0) mode.

Differences:

AspectUser ModeKernel Mode
AccessRestrictedFull
Hardware AccessNoYes
Privileged InstructionsNot allowedAllowed
Fault ImpactApp crashSystem crash

One-Liner Takeaway:

Kernel mode protects the OS while user mode ensures safe app execution.

Interview Focus 🔍:

  • Mode switching, purpose of dual-mode operation, mode bit.

5️⃣ Scheduling & Scheduling Queues ⏱️

Definition:

  • Scheduling determines process execution order to optimize CPU and resources.

Queues:

  • Job Queue: All processes in system.
  • Ready Queue: Processes in memory waiting for CPU.
  • Device Queue: Processes waiting for I/O devices.

Schedulers:

  • Long-Term: Controls multiprogramming.
  • Medium-Term: Swaps processes to manage memory.
  • Short-Term: Chooses ready processes for CPU.

Analogy: ✈️

  • Airport queues: Job = arriving passengers, Ready = waiting at gates, Device = waiting for services. Staff = schedulers.

One-Liner Takeaway:

Scheduling ensures fair CPU allocation and smooth system operation.

Interview Focus 🔍:

  • Types of schedulers, scheduling queues, their purpose.

6️⃣ Process Scheduling Algorithms ⚡

Preemptive vs Non-Preemptive:

AspectPreemptiveNon-Preemptive
CPU InterruptYesNo
ResponsivenessHighLow
OverheadHigherLower

Preemptive Use: Real-time systems, interactive OS.

Non-Preemptive Use: Batch systems.

Factors Considered:

  • Priority, arrival time, burst time, CPU/I/O bound, deadlines.

One-Liner Takeaway:

Scheduling algorithms decide which process runs next, balancing fairness and efficiency.

Interview Focus 🔍:

  • Differences between preemptive/non-preemptive, examples.

7️⃣ CPU Scheduling Algorithms – Part 1 🖥️

  1. FCFS (First Come First Serve): Non-preemptive, easy to implement, may cause convoy effect.
  1. SJF (Shortest Job First): Preemptive/non-preemptive, minimizes waiting time, may cause starvation.
  1. SRTF (Shortest Remaining Time First): Preemptive version of SJF, reduces average turnaround time.

Analogy: 🛒

  • FCFS = queue at store, SJF = fastest orders first.

One-Liner Takeaway:

Choose scheduling based on system goals: throughput vs fairness vs responsiveness.

Interview Focus 🔍:

  • FCFS convoy effect, SJF starvation, SRTF preemption.

8️⃣ CPU Scheduling Algorithms – Part 2 🖥️

  1. HRRN: Prioritizes longest-waiting tasks, balances efficiency and fairness.
  1. Round Robin (RR): Fixed time slices, preemptive, good for interactive systems.
  1. Priority-Based (PBS): Executes based on priority, may cause starvation.
  1. Multilevel Queue (MLQ): Processes in different queues with policies.
  1. Multilevel Feedback Queue (MLFQ): Adaptive, moves processes between queues, complex to implement.

Analogy: 🎮

  • RR = turns in multiplayer game, PBS = hospital emergencies, MLQ/MLFQ = differentiated customer service.

One-Liner Takeaway:

Advanced scheduling improves fairness, responsiveness, and adaptability.

Interview Focus 🔍:

  • Difference between RR, PBS, MLQ, MLFQ, starvation prevention.

9️⃣ Threads in Operating Systems 🧵

Definition:

  • Threads = smallest unit of execution in a process, share resources but run independently.

Analogy: 🍽️

  • Chef (process) cooking multiple dishes (threads) concurrently.

Types:

  • User-Level Threads (ULTs): Managed in user space, fast context switch, limited multiprocessor use.
  • Kernel-Level Threads (KLTs): Managed by OS, true concurrency, higher overhead.

Advantages:

  • Faster context switching, resource sharing, improved CPU utilization, scalability.

Disadvantages:

  • Complex debugging, synchronization overhead, risk of deadlocks.

Applications: Web servers, video streaming, interactive apps.

One-Liner Takeaway:

Threads enable concurrent execution within a process, enhancing efficiency and responsiveness.

Interview Focus 🔍:

  • Threads vs processes, ULT vs KLT, concurrency benefits.

🔟 User-Kernel Thread Models

Thread Mapping Models:

  1. Many-to-One: Multiple ULT → single KLT. Fast, but blocks entire process.
  1. One-to-One: Each ULT → separate KLT. True parallelism, higher overhead.
  1. Many-to-Many: Multiple ULT → smaller/equal KLT pool. Balanced concurrency, complex.

Comparison Table:

AspectULTKLT
Context SwitchFast, user spaceSlower, kernel calls
BlockingOne blocks allIndependent threads run
SchedulingUser libraryOS kernel
ConcurrencyLimitedTrue parallelism
ComplexitySimplerOS-level complexity

One-Liner Takeaway:

Thread models define trade-offs between speed, concurrency, and OS overhead.

Interview Focus 🔍:

  • Thread models (1:1, m:n, m:1), advantages/disadvantages, real-life usage.

✅ Key Takeaways – Module 2

  • Processes are the active programs, threads are lightweight execution units.
  • PCB is essential for process management and context switching.
  • Dual-mode (Kernel/User) ensures security and stability.
  • Schedulers and queues optimize CPU and resource utilization.
  • Preemptive vs Non-preemptive and CPU scheduling algorithms manage fairness, throughput, and response time.
  • Threads and their mapping models determine concurrency and parallelism in modern systems.
