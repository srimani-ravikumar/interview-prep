1. Cache Mapping Techniques üñ•Ô∏è

Definition:

A cache is a small, fast memory component between CPU and main memory that stores frequently accessed data to reduce average access time. Cache mapping determines which memory block goes into which cache line.

Why Cache Mapping Matters üîç

  • Efficient memory block location in cache.
  • Determines which cache line a memory block occupies.
  • Reduces cache misses and improves performance.

Types of Cache Mapping

Direct Mapping

  • Each memory block maps to exactly one cache line using modulus operation.
  • Analogy: A specific type of book always goes in one pre-assigned shelf compartment.
  • Advantages:
    • Simple implementation. ‚úÖ
    • Fast access to cache line. ‚ö°
  • Disadvantages:
    • High conflict misses. ‚ùå
    • Less flexible if multiple blocks map to the same line.

Associative Mapping

  • Any memory block can be placed in any cache line.
  • Analogy: Books can go anywhere on a flexible shelf.
  • Advantages:
    • Fewer conflict misses. ‚úÖ
    • Flexible memory access patterns.
  • Disadvantages:
    • Complex hardware. ‚öôÔ∏è
    • Slightly longer hit times.

Direct vs. Associative Mapping

FeatureDirect MappingAssociative Mapping
PlacementFixed line per blockAny line can store block
SpeedFast indexingSlower due to multiple line search
Conflict MissHighLow

One-liner takeaway: Direct mapping is fast but rigid; associative mapping is flexible but complex.

2. Virtual Machines (VMs) & Hypervisors üåê

Definition:

  • VM: Isolated environment simulating a full computer, running independently on shared physical hardware.
  • Hypervisor: Software/firmware managing VMs and allocating resources.

Analogy: Apartment building (host) ‚Üí apartments (VMs), manager (hypervisor) assigns electricity, water, and space.

Key Concepts

  • Hardware Abstraction: Hypervisor presents virtual hardware to VMs.
  • Resource Allocation: CPU, memory, storage divided per VM.
  • Isolation: One VM crash does not affect others.
  • Concurrent Execution: Multiple VMs run simultaneously.

Advantages

  • Efficient hardware utilization. ‚úÖ
  • Easy deployment & testing. ‚ö°

Disadvantages

  • Performance overhead. ‚è±Ô∏è
  • Resource contention. ‚ö†Ô∏è

5. Thread Safety & Reentrancy üßµ

Definitions:

  • Thread Safety: Code behaves correctly when accessed by multiple threads concurrently.
  • Reentrancy: Function can be safely re-entered before previous call completes.

Analogy:

  • Thread-safe ‚Üí Shared coffee machine handles multiple employees without errors.
  • Reentrant ‚Üí Recipe that multiple cooks can follow independently without interference.

Characteristics

  • Thread-safe: Synchronization, atomic operations, consistent state.
  • Reentrant: No global/static state, uses local data, deterministic.

Key Differences

FeatureThread-SafeReentrant
ScopePrevents concurrent data corruptionEach call independent
Shared StateMay use synchronized shared stateAvoids global state entirely
ComplexityLocks, semaphoresAvoid global/static data

Techniques to Achieve Both

  • Avoid global state
  • Use immutable data
  • Fine-grained synchronization
  • Stateless functions
  • Careful memory management

Advantages: Reliability, scalability, reusability.

Disadvantages: Complexity, performance overhead, maintenance challenge.

Interview focus üîç: Concurrency, mutexes, semaphores, reentrant vs thread-safe examples.

6. OS Security Mechanisms üîí

Definition:

Tools, policies, and controls in OS to protect resources from unauthorized access.

Analogy: Guarded building ‚Üí passwords, encryption, monitoring control access.

Key Concepts

  • Authentication: Verify user identity.
  • Authorization: Define user actions.
  • Access Control: MAC, DAC, RBAC.
  • Auditing & Logging: Track activities.
  • Encryption & Data Protection
  • Intrusion Detection & Prevention (IDPS)
  • Sandboxing & Isolation

Security Models

  • Bell-LaPadula: Confidentiality.
  • Biba: Data integrity.
  • Clark-Wilson: Transaction integrity.

OS Implementations

  • Linux: SELinux, AppArmor
  • Windows: UAC, BitLocker, Defender
  • macOS: Gatekeeper, FileVault
  • Mobile: Sandbox apps, permission frameworks

Challenges

  • Performance overhead
  • Configuration complexity
  • Zero-day attacks
  • User negligence

Takeaway: OS security mechanisms protect data, maintain integrity, and enforce user trust.

7. Future Trends in OS üåü

Microkernels

  • Minimal kernel ‚Üí only essential services.
  • Other services run in user space.
  • Advantages: Security, stability, maintainability.
  • Disadvantages: Increased IPC overhead.

Analogy: Restaurant split into specialized stations ‚Üí faults isolated.

Containers

  • Lightweight virtualization packaging app + dependencies.
  • Share host kernel, isolated filesystem & libraries.
  • Advantages: Lightweight, fast startup, portable.
  • Challenges: Host kernel compromise, orchestration complexity.

Analogy: Shipping containers ‚Üí easy, standardized, isolated transport.

Other Trends

  • Unikernels ‚Üí single-purpose OS images.
  • Cloud-native OS ‚Üí optimized for distributed workloads.
  • Serverless ‚Üí abstract server management.
  • Edge & IoT OS ‚Üí lightweight for constrained devices.

Takeaway: OS evolution emphasizes modularity, isolation, scalability, and portability.
