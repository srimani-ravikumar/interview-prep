üóÉÔ∏è 3. File Access Methods

üîç Access Types

  1. Sequential Access:
    • Data accessed in order (like reading a tape).
    • Example: Text files.
  1. Direct Access:
    • Access by byte/block number (like random song playback).
    • Example: Databases.
  1. Indexed Access:
    • Index points to data blocks, enabling faster retrieval.
    • Example: Index-based retrieval in a phonebook app.

‚ö° Takeaway

Choice of access method impacts speed and flexibility of file operations.

üîç Interview Focus

  • Difference between sequential and direct access.

üìÇ 4. Directory Structure

üèóÔ∏è Directory Organization

  • Single-Level: One directory for all files (simple, but name conflicts).
  • Two-Level: Separate directory for each user.
  • Tree-Structured: Hierarchical (most common).
  • Acyclic Graph: Allows shared subdirectories.
  • General Graph: Supports links but may cause cycles.

üß© Directory Operations

  • Create/Delete
  • Search
  • Rename
  • Traverse
  • Display

‚ö° Takeaway

Directory structures define how efficiently files can be organized and retrieved.

üß± 6. Free Space Management

üìä Techniques

  • Bit Vector: 1 for used, 0 for free blocks.
  • Linked List: Free blocks linked together.
  • Grouping: Stores addresses of free blocks in clusters.
  • Counting: Tracks starting address and count of consecutive free blocks.

‚ö° Takeaway

Efficient free space management avoids wasted storage and fragmentation.

üîç Interview Focus

  • How does the OS track free space on disk?
